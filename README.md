# Eco Image Optimizer

Eco Image Optimizer is a simple browser-only React app that uploads one image to Cloudinary, shows the original and optimized previews, and estimates the bandwidth and CO2 savings from Cloudinary optimization.

The app uses Cloudinary unsigned uploads directly from the browser. There is no backend.

Live app: https://eco-image-optimizer.vercel.app

## Features

- Upload one image from the browser
- Preview the original selected image
- Upload to Cloudinary using an unsigned upload preset
- Preview the optimized Cloudinary image
- Apply Cloudinary `f_auto,q_auto` transformations
- Display original file size
- Estimate optimized file size
- Calculate bandwidth saved
- Calculate percentage reduction
- Estimate CO2 saved
- Clearly labels CO2 savings as an estimate
- Responsive CSS for desktop and mobile screens

## Tech Stack

- React
- Vite
- Cloudinary unsigned browser uploads
- Vercel hosting

## How Optimization Works

After a user selects an image, the app uploads it to Cloudinary with an unsigned upload preset. Cloudinary returns a hosted image URL.

The optimized preview URL is created by inserting the following transformation into the Cloudinary upload URL:

```txt
f_auto,q_auto
```

These Cloudinary transformations mean:

- `f_auto`: Cloudinary automatically serves the best image format for the browser, such as WebP, AVIF, PNG, or JPEG.
- `q_auto`: Cloudinary automatically chooses a quality level that balances image quality and file size.

## CO2 Estimate

The app uses a deliberately simple estimate:

```txt
0.81 grams CO2 per MB transferred saved
```

The CO2 value shown in the app is an estimate, not a lifecycle assessment. It is intended as a lightweight way to communicate the rough environmental benefit of reducing transferred image bytes.

## Environment Variables

Create a local `.env` file based on `.env.example`:

```bash
VITE_CLOUDINARY_CLOUD_NAME=your_cloud_name
VITE_CLOUDINARY_UPLOAD_PRESET=your_unsigned_upload_preset
```

These variables are exposed to the browser because they use Vite's `VITE_` prefix. That is expected for this app. Do not put Cloudinary API secrets in this project.

For Vercel, add the same variables in the project settings:

- `VITE_CLOUDINARY_CLOUD_NAME`
- `VITE_CLOUDINARY_UPLOAD_PRESET`

## Cloudinary Setup

1. Create or open a Cloudinary account.
2. Create an unsigned upload preset.
3. Make sure the preset allows image uploads from the browser.
4. Copy your Cloudinary cloud name into `VITE_CLOUDINARY_CLOUD_NAME`.
5. Copy the unsigned preset name into `VITE_CLOUDINARY_UPLOAD_PRESET`.

Only unsigned upload configuration is needed. No backend endpoint or API secret is required.

## Local Development

Install dependencies:

```bash
npm install
```

Start the development server:

```bash
npm run dev
```

Build for production:

```bash
npm run build
```

Preview the production build locally:

```bash
npm run preview
```

## Deployment

The app is configured for Vercel with `vercel.json`:

```json
{
  "buildCommand": "npm run build",
  "outputDirectory": "dist",
  "framework": "vite"
}
```

When the GitHub repository is connected to Vercel, Vercel can automatically build and deploy changes pushed to the main branch.
